Working in the e-commerce industry, I had been made aware of the shipping container shortage cause by Covid-19 in 2021. Being Panamanian, and knowing that maritime transport is one of the pillars of Panamanian economy, I can't help but wonder how ports back home are "faring".

Fortunately, I was able to find this by the Panamanian Maritime Authority on container traffic, both empty and full, measured TEU's (twenty-foot equivalent units) processed every month in 2019 through 2021.

After looking at the table for a while and seeing there were only six ports to track over three years, a bump chart seemed like a good representation. In addition to how each port ranked among all ports for a time period, I decided to add another encoding: How each time period compared to others for any given port by varying the amplitude of the ocean waves ðŸŒŠ. This required changing the traditional bump chart so that I wasn't referring to points in time, but ranges instead.


My Process

The vision was really straight forward, an enhanced bump chart that used "wave amplitude" to convey the ranking within each of the categories. I first did a pen-and-paper sketch. The positions and shape of the waves were absolutelu made up, but it gave me a reference that I could now attempt to duplicate using D3.


After I had a basic ideo of what the final chart would look like, there were three main phases that went into constructing this visualization:

The first one was building the wavy bump chart in a grid, where there would be one row per port and each column would represent a time period (quarters in this case). Because I wanted to emphasize ranking, like in a bump chart, instead for actual values of the vertical axis, I decided to plot each point twice: once at the beginning and once at the end of each time period. This meant that instead of points, I was connecting horizontal line segments. In addition to this, I used a "wave amplitude" to encode the ranking of each quarter in comparison to other quarters for each port. To do this, I simply used the repeating values of a sine wave (0,1,0,-1).


I defined a fixed amplitude of a few pixels and started with a straight line that was really just a horizontal line segments connected to each other. For example, the following positions: (0,0), (10,0), (20,0), (30,0), etc. I used 9 points for each - 8 becase I wanted the sine wave to repeat twice so it could be appreiated better, and an extra point because each segment needs a start and an end. Then I scaled the sine sequence by this amplidute, for example (0, 10, 0, -10), and I added this repeating scaled pattern to my line segment y-coordinates so the vertical value would shift up and down like a wave. This was relatively easy with the d3.line().curve() function to generate an SVG path for each port.


The second part of the process was making the wave amplitudes vary each time period. This meant that instead of using a constant value, the scaling value for the y-coordinate of each point would be calculated depending on where that specific quarter ranked for the port. This meant that now the visualization would get messier, but at this point playing with the stroke widths and color could make it more aesthetically pleasing. At least the lines were not squiggly for no reason, there was actual qualitative information.


The last step was mostly cosmetic and consisted in adding annotations, such as a title, a legend, and styling the axes (adding port names and formatting ticks). Finally, a drop-down list for user input to change the subset of the data from processing of empty, full, or all containers. While there are definitely more robust methods to do this, Observable views came in handy for adding this interactive detail!
